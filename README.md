## Лабораторная работа №22. Интерактивные веб-приложения на JavaScript

**Цель лабораторной работы**:
Закрепить и систематизировать знания по JavaScript путём разработки интерактивных веб-приложений с использованием HTML, CSS и JavaScript, а также закрепить навыки работы с системой контроля версий Git и удалёнными репозиториями GitHub

**Описание работы:**
Данный проект представляет собой веб-приложение "Калькулятор". Калькулятор выполнен на чистом JavaScript с использованием современных стандартов ES6+ и модульной архитектуры. Приложение поддерживает базовые арифметические операции, валидацию ввода и обработку ошибок.

## Функциональность:

### Поддерживаемые операции:
- Сложение (`+`)
- Вычитание (`-`)
- Умножение (`*`)
- Деление (`/`)
- Десятичные дроби (`.`)
- Очистка поля (`C`)

### Ограничения:
- Максимальная длина выражения не ограничена, но зависит от возможностей браузера
- Деление на ноль обрабатывается как ошибка
- Невозможно ввести несколько операторов подряд (валидация заменяет последний оператор)
- Невозможно начать выражение с точки (`.`)
- Невозможно ввести более одной точки в текущем числе

### Обработка ошибок:
- **Некорректное выражение**: при попытке вычислить выражение с синтаксическими ошибками (например, `5++3`) отображается сообщение "Ошибка"
- **Деление на ноль**: перехватывается и отображается как ошибка
- **Некорректный результат**: при получении бесконечности (`Infinity`) или нечислового значения (`NaN`) отображается ошибка
- **Состояние ошибки**: после возникновения ошибки калькулятор блокирует дальнейший ввод до нажатия кнопки `C`

**Логика работы:**

### Вычисление результата:

Процесс вычисления в методе `calculate()`:

1. Проверка наличия выражения и отсутствия ошибки
2. Вызов `safeEvaluate()` для безопасного вычисления
3. В `safeEvaluate()`:
   - Проверка корректности выражения через `isValidExpression()`
   - Вызов `eval()` для вычисления (изолированно)
4. Проверка результата на конечность (`isFinite()`)
5. Обновление `expression` результатом вычислений
6. Обновление дисплея

**Почему используется `eval()`?**
В учебных целях `eval()` применяется для простоты реализации. В production-коде следует использовать парсер математических выражений или библиотеки типа `math.js`.

### Валидация выражения:

Метод `isValidExpression(expr)` проверяет:

1. **Двойные операторы**: регулярное выражение `/[+\-*/]{2,}/` проверяет отсутствие двух и более операторов подряд
2. **Оператор в конце**: выражение не должно заканчиваться оператором

Примеры валидных выражений:
- `5+3`
- `10/2`
- `3.5*2`

Примеры невалидных выражений:
- `5++3` (двойной оператор)
- `5+` (оператор в конце)
- `5*/3` (недопустимая комбинация операторов)
